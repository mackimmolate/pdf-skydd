# Security Policy

## Scope

PDF-skydd is a local browser-based tool for adding an open password to PDF
files. Security issues are taken seriously when they affect:

- password handling
- accidental file disclosure or network transfer
- generation of broken or unsafe PDF output
- dependency or build-chain compromise that impacts shipped behavior

## Reporting

Do not open a public issue for security-sensitive findings.

Use GitHub's private vulnerability reporting for this repository if it is
enabled. If it is not enabled, contact the maintainer privately through GitHub
with:

- the affected version, commit, or deployment date
- browser and operating system
- clear reproduction steps
- expected behavior and actual behavior
- impact assessment
- sample files only if they are safe to share

## Response

Security reports are handled on a best-effort basis. There is no guaranteed
response SLA for this project.

The focus is on confirming the issue, fixing the current supported version, and
documenting any user action required after the fix.
